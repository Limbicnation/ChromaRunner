# Plan: Fix PingPong / Loop Patrol Traversal in EnemyCharacter

## Problem Analysis

The enemy patrol system has **two separate patrol paths**: simple (origin-based) and waypoint-based. Both have broken or missing PingPong/Loop behavior.

### Bug 1: Waypoint patrol has NO pause at endpoints
`PatrolStepWaypoint()` (line 326) calls `AdvanceWaypointIndex()` immediately on arrival — the enemy reverses/wraps instantly with zero pause. The `PauseAtEndpoint()` function is **only called from `PatrolStepSimple()`**, never from the waypoint path.

### Bug 2: `ResumePatrol()` calls `StartPatrol()`, which resets all state
`StartPatrol()` (line 266) resets `CurrentNodeIndex = 0`, `bWaypointReverse = false`, `PatrolDirection = 1.0f`. After any pause the enemy teleports its patrol index back to the start instead of continuing from where it was.

### Bug 3: Simple patrol ignores `TraversalMode`
`PatrolStepSimple()` (line 303) always PingPongs — it never checks `TraversalMode`. When set to `Loop`, the simple patrol should wrap back to `PatrolOrigin` instead of reversing direction.

### Bug 4: Waypoint Loop mode has no pause at wrap point
`AdvanceWaypointIndex()` Loop branch (line 405) wraps `CurrentNodeIndex` via modulo with no pause, making the wrap feel like a sudden teleport.

## Step-by-Step Changes

### Step 1 — `EnemyCharacter.h`: Add pause state and resume method

Add to private section:
```cpp
bool bIsPaused = false;
void ResumePatrolStep();
```

### Step 2 — `EnemyCharacter.cpp`: Create `ResumePatrolStep()`

New method that restarts the patrol timer **without** resetting state:
```cpp
void AEnemyCharacter::ResumePatrolStep()
{
    if (bIsDead) return;
    bIsPaused = false;

    // Restore walk flipbook
    if (EnemySprite && WalkFlipbook)
    {
        EnemySprite->SetFlipbook(WalkFlipbook);
    }

    bIsPatrolling = true;
    GetWorldTimerManager().SetTimer(
        PatrolTimerHandle,
        this,
        &AEnemyCharacter::PatrolStep,
        PATROL_STEP_INTERVAL,
        true
    );
}
```

### Step 3 — `EnemyCharacter.cpp`: Refactor `PauseAtEndpoint()` for both systems

Make `PauseAtEndpoint` generic — it pauses the timer and starts a one-shot that calls `ResumePatrolStep()`:
```cpp
void AEnemyCharacter::PauseAtEndpoint()
{
    bIsPatrolling = false;
    bIsPaused = true;
    GetWorldTimerManager().ClearTimer(PatrolTimerHandle);

    if (EnemySprite && IdleFlipbook)
    {
        EnemySprite->SetFlipbook(IdleFlipbook);
    }

    GetWorldTimerManager().SetTimer(
        PauseTimerHandle,
        this,
        &AEnemyCharacter::ResumePatrolStep,
        PatrolPauseTime,
        false
    );
}
```

### Step 4 — `EnemyCharacter.cpp`: Fix `ResumePatrol()` to NOT call `StartPatrol`

Update `ResumePatrol()` to delegate to `ResumePatrolStep()` instead of `StartPatrol()`. Or simply make `ResumePatrol` call `ResumePatrolStep()`. This is only called from the old simple patrol path, so it needs the `PatrolDirection *= -1.0f` flip before resuming.

```cpp
void AEnemyCharacter::ResumePatrol()
{
    if (bIsDead) return;
    PatrolDirection *= -1.0f;  // Flip direction (simple patrol)
    ResumePatrolStep();
}
```

Wait — actually `PauseAtEndpoint` already flips direction (line 427). Let me re-examine...

Current flow: `PauseAtEndpoint()` does `PatrolDirection *= -1.0f` then sets timer to `ResumePatrol`. `ResumePatrol()` calls `StartPatrol()` which resets state.

**Better approach**: Remove the `PatrolDirection *= -1.0f` from `PauseAtEndpoint()` (since `AdvanceWaypointIndex` handles direction), and have simple patrol do the flip in its own resume path.

Actually, simplest fix: keep `PauseAtEndpoint()` doing the flip (only used by simple patrol now), and have it call `ResumePatrolStep()`. Then make `ResumePatrol()` a simple wrapper that does the flip + `ResumePatrolStep()`.

**Revised approach:**
- `PauseAtEndpoint()` — generic pause (no direction flip). Used by both patrol systems.
- `ResumePatrol()` — simple patrol resume. Flips `PatrolDirection`, then calls `ResumePatrolStep()`.
- `ResumePatrolStep()` — restarts timer without resetting state.

### Step 5 — `EnemyCharacter.cpp`: Add pause to waypoint PingPong at endpoints

In `AdvanceWaypointIndex()`, when the enemy reaches the first or last waypoint in PingPong mode, call `PauseAtEndpoint()` before updating the index:

```cpp
if (TraversalMode == EPatrolTraversalMode::PingPong)
{
    if (!bWaypointReverse)
    {
        if (CurrentNodeIndex >= PatrolWaypoints.Num() - 1)
        {
            bWaypointReverse = true;
            PatrolDirection = -1.0f;
            PauseAtEndpoint();  // Pause before reversing
            CurrentNodeIndex = FMath::Max(0, PatrolWaypoints.Num() - 2);
            return;
        }
        ++CurrentNodeIndex;
    }
    else
    {
        if (CurrentNodeIndex <= 0)
        {
            bWaypointReverse = false;
            PatrolDirection = 1.0f;
            PauseAtEndpoint();  // Pause before reversing
            CurrentNodeIndex = FMath::Min(1, PatrolWaypoints.Num() - 1);
            return;
        }
        --CurrentNodeIndex;
    }
}
```

## Summary of Behavior After Fix

| Mode | Patrol Type | Endpoint Behavior |
|------|-----------|-------------------|
| PingPong | Simple | Pause → reverse `PatrolDirection` → resume |
| PingPong | Waypoint | Pause → set `bWaypointReverse` → resume from adjacent node |
| Loop | Simple | Snap back to `PatrolOrigin` → continue forward (no pause) |
| Loop | Waypoint | Pause at wrap point → resume from index 0 |
